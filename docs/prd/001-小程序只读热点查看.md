# 001 小程序只读热点查看

## 功能目标

小程序 V1 作为只读轻量端，帮助用户快速查看今日热点、事件详情、来源摘要、Obsidian 同步状态和日报状态。所有配置仍在 Web 工作台完成。

## 用户故事

1. 作为用户，我可以在手机上查看今日热点列表。
2. 作为用户，我可以打开事件详情查看摘要、评分、来源和时间线。
3. 作为用户，我可以看到 Obsidian 同步和邮件日报是否成功。
4. 作为用户，我知道需要配置或修复授权时应回到 Web 工作台。

## 范围

1. 登录或会话绑定。
2. 今日热点列表。
3. 事件详情只读展示。
4. 同步状态和日报状态只读展示。
5. 失效授权和配置缺失提示。

## 非范围

1. 平台授权配置。
2. 主题创建或编辑。
3. Git 设置。
4. 邮件订阅设置。
5. 删除和撤权操作。

## 状态模型

小程序展示状态必须来自 server：`no_topic`、`collecting`、`event_ready`、`sync_failed`、`mail_failed`、`auth_required`、`server_unavailable`。

## 契约影响

小程序客户端类型必须由 server OpenAPI 生成。V1 可复用 Web 的事件列表、事件详情和状态接口，但不得依赖 Web 专用聚合接口。

## SDD 要求

1. 先定义只读页面和接口依赖。
2. 先定义空态、错误态和跳转 Web 的提示。
3. 先定义小程序端可执行验收方式。
4. 先定义与 Web 状态文案的一致性规则。

## TDD 要求

1. 先写页面渲染测试或等价组件测试。
2. 先写接口 mock 状态覆盖测试。
3. 先写无主题、授权失效、同步失败和服务不可用测试。
4. OpenAPI 变更后先生成客户端。

## E2E 验收门禁

1. 构建通过并产出小程序包。
2. 自动化验收覆盖今日热点列表和事件详情。
3. 自动化验收覆盖无主题、授权失效和同步失败状态。
4. 小程序只读端不出现配置、授权或删除入口。

## 数据与权限

1. 小程序只读取当前用户数据。
2. 不展示敏感 token、内部错误堆栈或原始平台凭据。
3. 登录失效时进入重新登录状态。
